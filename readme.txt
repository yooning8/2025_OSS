■ 데이터베이스 복구 방법
제공된 `rewear_dump.sql` 파일을 사용하여 MySQL에서 다음과 같이 데이터베이스를 복원할 수 있습니다:

1. re_wear_db 데이터베이스 생성
   CREATE DATABASE re_wear_db;

2. MySQL에 접속 후, 아래 명령 실행
   SOURCE rewear_dump.sql;

   또는 CLI에서:
   mysql -u root -p re_wear_db < rewear_dump.sql

이 과정을 통해 users, center 테이블이 생성되며, 테스트 계정(system)도 함께 포함됩니다.
비밀번호: dbsl2023!

■ 1. 실행 준비
1. 프로젝트 압축 파일을 해제합니다.
2. IntelliJ에서 해당 폴더를 열어 프로젝트를 실행합니다.
3. ReWearApplication.java에서 우클릭 → Run 을 통해 서버를 실행합니다.
4. 실행 후 브라우저 주소창에 아래 주소를 입력합니다:
   → http://localhost:8080/category.html

■ 2. 계정 정보
- 최초 접속 시 로그아웃 상태이며, 사용할 수 있는 기능은 [기부처 목록 조회]와 [로그인] 버튼뿐입니다.
- 다음 중 하나를 선택하여 로그인할 수 있습니다:
  ① 직접 회원가입 (signup.html)
  ② 관리자 계정 사용 (이름만 관리자이며 권한 제한 있음)
     - ID: system
     - PW: system1234

■ 3. 주요 기능 안내

[메인화면: category.html]
- 로그인 후 다양한 기능 사용 가능
  - 기부하기
  - 기부 현황 조회
  - 기부처 목록 보기
  - 마이페이지

[기부하기: donate.html]
- 기부 절차는 5단계로 구성됩니다. 순서를 건너뛸 수 없도록 설계되어 있습니다.
  1) 박스 배송 요청
     - 기부할 박스 수 입력 (숫자만 가능)
     - 입력 완료 시 체크박스가 파란색으로 활성화됨 (주의: 외곽선 주의 깊게 보기)
  2) 주소 입력
     - 주소 입력 후 '주소 입력 완료' 버튼 클릭
  3) 포장 완료
     - 실제 포장이 아닌, 버튼 클릭으로 다음 단계 활성화
  4) 기부처 선택
     - 등록된 기부처 목록에서 선택
  5) 기부 완료
     - 모든 절차 완료 후 기부 정보가 localStorage에 저장됨

[기부 현황 조회: status.html]
- 현재 로그인한 계정의 기부 내역을 확인할 수 있습니다.

[기부처 목록 보기: centers.html]
- 등록된 기부처의 이름, 주소, 전화번호 확인 가능
- 사용자가 직접 새로운 기부처를 등록할 수도 있습니다.
- 등록된 기부처는 삭제 후 재등록이 가능합니다.

[마이페이지: mypage.html]
- 현재 로그인한 계정의 정보를 확인 및 수정할 수 있습니다.
  (예: 주소 변경, 이름 수정 등)

■ 4. 주의사항
- 모든 HTML 파일은 static 폴더에 위치해 있으며, 템플릿 엔진 없이 정적으로 동작합니다.
- 데이터베이스에 저장되는 정보는 회원(User), 기부처(Center)이며,
  기부 내역은 현재 localStorage에 저장됩니다.
